Visão Geral

A Tábua de Marés API fornece dados precisos sobre as marés do litoral brasileiro. Uma API REST completa para desenvolvedores, pesquisadores e entusiastas do mar.

🌐 Base URL: ${url_env}/api/{{apiVersion}}

📊 Formato: JSON

🚀 Status: Ativo (Recomendado)Depreciado

✨ Características

  • 🌊 Cobertura completa: Dados de marés para todo o litoral brasileiro
  • 📍 17 estados costeiros: Informações detalhadas por localização
  • ⚡ Respostas rápidas: API otimizada para performance
  • 🔒 Confiável: Dados oficiais e atualizados

Autenticação

🎉 Uso Livre: A API é de uso público. O plano free não requer autenticação (limite por IP).

⚡ Limites (free, por IP): 24 requisições por minuto e 32.000 requisições por mês.

🔐 Planos pagos (por api_key):

  • R$ 5/mês: 128 req/min + 256.000 req/mês
  • R$ 10/mês: 256 req/min + ilimitado req/mês

Envie a api_key no header Authorization: Bearer <api_key> ou X-Api-Key.

🔒 Login Google: Faça login em /auth/google para gerenciar sua conta e api_keys.

⚠️ V1 Depreciada: A V1 foi descontinuada e responde 410 Gone. Use a V2. Veja as diferenças abaixo.

curl -X GET "${url_env}/api/{{apiVersion}}/harbor_names/pb"

Obter Portos por IDs

GET

⚠️ Depreciado: A V1 utiliza IDs numéricos do porto vinculados diretamente ao banco de dados. Recomendamos a migração para a V2, que utiliza IDs baseados no estado (ex: 'pb01') referenciados ao ano corrente, tornando a integração mais estável e segura!

Retorna informações detalhadas de um/mais portos específico pelo seu ID.

GET /api/{{apiVersion}}/harbors/{ids}

📋 Parâmetros:

  • ids (string): IDs dos portos (ex: [pb01, pe02]) (integer): IDs numéricos (ex: [1,2,3])

📝 Exemplo de Requisição:

# Obter informações do porto pb01
curl -X GET "${url_env}/api/{{apiVersion}}/harbors/[pb01]"# Obter informações do porto ID 1
curl -X GET "${url_env}/api/{{apiVersion}}/harbor/[1]"

Obter Tábua de Maré por Período

GET

⚠️ Depreciado: A V1 utiliza IDs numéricos do porto legados. Na V2, os IDs são estruturados por estado e ano corrente, garantindo maior segurança e consistência nos dados.

Retorna os dados da tábua de maré para um porto específico em um mês e dias determinados.

GET /api/{{apiVersion}}/tabua-mare/{harbor}/{month}/{days}

📋 Parâmetros:

  • harbor (string): ID do porto (ex: 'pb01')(integer): ID numérico (ex: 1)
  • month (integer): Mês desejado (1-12)
  • days (string): Array de dias no formato "[1,2,10-30]", dias específicos separados por vírgula (ex: 1,30,27) e/ou um período (ex: 1,5-13).

📝 Exemplo de Requisição:

# Obter tábua de maré do porto pb01
curl -X GET "${url_env}/api/{{apiVersion}}/tabua-mare/pb01/1/[1,2,3]"# Obter tábua de maré do porto ID 1
curl -X GET "${url_env}/api/{{apiVersion}}/tabua-mare/1/1/[1,2,3]"

✅ Exemplo de Resposta:

{{code_resposta_tabua_mare}}

Porto mais próximo (estado)

GET

Retorna o porto mais próximo das coordenadas informadas dentro do estado especificado.

GET /api/{{apiVersion}}/nearested-harbor/{state}/{lat_lng}

📋 Parâmetros:

  • state (string): Sigla do estado (ex: pb, rj, sp).
  • lat_lng (string): Coordenadas no formato [lat,lng].

📝 Exemplo de Requisição:

curl -X GET "${url_env}/api/{{apiVersion}}/nearested-harbor/pb/[-7.11509,-34.864]"

🧪 Exemplo com Fetch:

fetch(`${url_env}/api/{{apiVersion}}/nearested-harbor/pb/[-7.11509,-34.864]`).then(r => r.json())

Tábua de Maré por Geolocalização

GET

Obtém a tábua de maré do porto mais próximo dentro de um estado, informando apenas coordenadas e período.

GET /geo-tabua-mare/{lat_lng}/{state}/{month}/{days}

📋 Parâmetros:

  • lat_lng (string): Coordenadas [lat,lng].
  • state (string): Sigla do estado (ex: pb, rj, sp).
  • month (integer): Mês (1-12).
  • days (string): Dias e/ou intervalos (ex: [1,2,10-30]).

📝 Exemplo de Requisição:

curl -X GET "${url_env}/geo-tabua-mare/[-7.11509,-34.864]/pb/1/[1,2,3]"

🧪 Exemplo com Fetch:

fetch(`${url_env}/geo-tabua-mare/[-7.11509,-34.864]/pb/1/[1,2,3]`).then(r => r.json())

⚙️ Detalhes técnicos

  • lat_lng é parseado no servidor por um tipo dedicado e suporta negativos e decimais.
  • days aceita dias e intervalos; o servidor expande e deduplica valores.
  • O porto é escolhido pela menor distância às coordenadas, respeitando o estado.
